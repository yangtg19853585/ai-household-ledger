# AI 가계부 (MVP 데모)

카드 알림을 자동(또는 공유)으로 앱에 전달하면, AI가 가맹점을 분석해 자동으로 카테고리를 분류해주는 모바일 가계부 프로토타입입니다. 플랫폼별로 알림을 가져오는 방식이 달라, **iOS 버전**과 **Android 버전** 두 가지 데모를 각각 제공합니다.

## 데모 링크

- **iOS**: https://yangtg19853585.github.io/ai-household-ledger/
- **Android**: https://yangtg19853585.github.io/ai-household-ledger/android.html
- **이용 가이드**: https://yangtg19853585.github.io/ai-household-ledger/guide.html — 처음 써보는 사람을 위한 사용법 안내

각 페이지 상단의 "Android →" / "iOS →" / "📘 가이드" 링크로 서로 이동할 수 있어요. 폰 브라우저(사파리/크롬)로 열어서 "홈 화면에 추가"를 하면 앱처럼 아이콘을 눌러 실행할 수 있습니다.

## 두 버전의 차이 — 왜 다르게 만들었나요

같은 목표("카드 결제 알림을 읽어 자동으로 가계부 분류")를 두 플랫폼이 서로 다른 방식으로만 구현할 수 있어서, UX 자체가 달라집니다.

|  | iOS | Android |
|---|---|---|
| 알림을 가져오는 방법 | 사용자가 카드 승인 문자를 **공유하기**로 앱에 직접 전달 (Share Extension) | 앱이 **알림 접근 권한**(NotificationListenerService)으로 카드 알림을 자동으로 읽음 |
| 사용자 동작 | 문자 도착 → "공유하기" 탭 → 앱 선택 (수동 1단계 필요) | 알림 도착 → 끝 (사용자 동작 없음) |
| 이유 | iOS는 다른 앱의 알림/문자 내용을 읽는 공개 API가 없음 | Android는 알림 접근 권한으로 다른 앱 알림의 텍스트를 직접 받을 수 있음 |

두 데모 모두 이 흐름만 화면에서 시뮬레이션한 것이고, 실제 문자/알림 연동은 되어 있지 않습니다.

## 공통 기능

앱은 **홈 · 기록 · 테스트** 세 화면으로 나뉘어 있어요 (하단 탭으로 이동).

- **홈**: 기간을 직접 골라 볼 수 있는 총 지출, 예산 진행률, 카테고리별 막대그래프, 절약 인사이트, 가계부 내역. 실시간 데이터가 없으면 0원부터 시작합니다.
- **기록**: 2026년 1월~6월, 월 약 200만 원 상당의 더미 지출 내역이 미리 채워져 있어요. 월을 탭하면 홈 화면 기간이 그 달로 바뀌면서 총 지출/예산/차트가 그 달 기준으로 다시 표시돼요.
- **테스트**: 실제 서비스라면 없을, 카드 알림을 흉내 내는 시뮬레이션 화면. 처음 보는 가맹점이면 카테고리를 직접 선택하고, 다음부터는 자동 분류돼요. 편의점·쿠팡처럼 매번 카테고리가 다른 가맹점은 "기억 안 함"으로 남겨두면 매번 다시 물어봐요.
- **예산 & 절약 인사이트**: 홈 화면에서 총 예산과 카테고리별 예산을 설정할 수 있고, 예산을 넘기면 카드 알림 처리 직후 경고 배너가 떠요. "절약 인사이트" 카드는 예산 대비 사용률과, 예산이 없어도 지출이 한 카테고리에 몰려있으면 알려줘요.
- **이전 기간 대비 비교**: 홈 화면 총 지출 바로 아래에, 선택한 기간을 직전 같은 길이의 기간과 비교해 "이전 기간 대비 ▲12%" 식으로 보여줘요.
- **예산 초과 예측**: 이번 예산 주기가 아직 끝나지 않았고 지금까지는 예산 안에 있어도, 지금 페이스대로면 주기 말에 초과할 것 같으면 예산 카드에 미리 경고를 띄워요.
- **가계부 내역 검색**: "가계부 내역" 목록 위 검색창에 가맹점명이나 카테고리를 입력하면 그 기간 내역을 바로 필터링해요.
- 데이터는 브라우저 `localStorage`에 저장되어 새로고침해도 유지되고, iOS/Android 두 페이지가 같은 브라우저의 저장소를 공유합니다(같은 가계부라는 설정). "데모 데이터 초기화"는 테스트로 쌓은 내역·기간 설정·검색어만 지우고, 가맹점 분류 설정·예산 설정·기록 탭의 월별 데이터는 그대로 남아요.

## 한계 (MVP 범위)

- 현대카드 알림 포맷만 가정한 프로토타입이며, 실제 알림/문자 연동은 되어 있지 않습니다.
- 모든 데이터는 브라우저에만 저장되며, 서버나 실제 카드 데이터를 사용하지 않습니다.
- 실제 네이티브 앱(iOS Share Extension / Android NotificationListenerService) 구현은 다음 단계입니다.
